Unit tests

GEU utiliza Google Test como framework de testing. Por tanto, es necesario conocer como escribir un unit test utilizando este framework para escribirlo en GEU. Aquí se puede ver brevemente como escribirlos.

📘 Instrucciones

Cómo crear un unit test

Añadimos un fichero fuente que contenga el código del test en GEUTests/Source. Se recomienda añadirlo a un directorio que describa que parte de la aplicación está probando.

Esta es una plantilla base para cualquier unit test:

#include "GEUTestsPch.h"
#include "gtest/gtest.h"
//Dentro de este bloque se definen los tests.
TEST(TestCaseName, TestName) {
EXPECT_EQ(1, 1);
EXPECT_TRUE(true);
}

Cabe destacar que si queremos probar un módulo que no estuviera previamente añadido, se debe de añadir el nombre de dicho proyecto a la instrucción Links en el script de generación del proyecto Build-Tests.lua en el directorio GEUTests.

Cómo lanzar los Unit test

En la carpeta Project Scripts encontraremos un script llamado SetupTests.bat. Al ejecutarlo, se generará la solución para el testing. Cuando se compile y se ejecute dicha solución, se lanzarán todos los tests.

Si se quiere ejecutar tests de manera independiente, se recomienda la utilización de la extensión Resharper C++.
