Arquitectura de transferencia de datos en GEU

El sistema de GEU, principalmente se compone de tres componentes:

  • La base de datos (BBDD), la cual almacena los enlaces a los archivos ubicados dentro de la NAS
  • GEU, el cual hace de intermediario y trabaja con los archivos
  • NAS, la cual almacena los archivos como tal

Estos tres componentes están interconectados entre sí siguiendo la siguiente estructura:

graph TD
BBDD -- Manda enlace --> GEU
NAS -- Manda archivo --> GEU
GEU -- Pide enlace --> BBDD
GEU -- Pide archivo --> NAS

El funcionamiento de este sistema es que GEU pide a la base de datos el enlace de un archivo específico (dentro de la NAS). La base de datos devuelve a GEU el enlace del archivo y GEU, automáticamente, pide el archivo a la NAS enviándole el enlace recibido. Por último, la NAS devuelve el archivo como tal a GEU.

Hay que tener en cuenta que, para que esto funciona, en GEU se debe tener acceso y estar conectado tanto a la base de datos como a la NAS (los dos iconos deben aparecer en verde).
