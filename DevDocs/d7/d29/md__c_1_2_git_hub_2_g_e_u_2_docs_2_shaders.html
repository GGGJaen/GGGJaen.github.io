Shaders

El proyecto utiliza principalmente dos uber shaders (uno para las nubes y otro para las mallas de triángulos), donde se implementa toda la funcionalidad requerida para el renderizado. Cada funcionalidad se implementa en forma de subrutina por dos motivos:

  1. Se habilita la detección desde el código en C++ de las mismas.
  2. Permite diferentes implementaciones de la misma funcionalidad.

:blue_book: Instrucciones

  1. Se crea la subrutina en el shader. Lo único que debe de respetar es que el nombre del uniform que representa la subrutina tiene que tener el siguiente formato: nombreSubrutina + “Uniform”. Por ejemplo, si la subrutina es LightAttenuation, el uniform se llamará LightAttenuationUniform.
  2. Nos desplazamos al fichero ShaderEnums.h (Archivos de encabezado -> Core -> Shaders).
  3. Se deberá añadir a los enums de subrutinas un nombre identificativo.
  4. El nombre de la subrutina se debe incluir en SubroutineName, en la etapa correspondiente (VertexShader, Geometry Shader, Fragment Shader o Compute Shader).
  5. El nombre de cada implementación de la subrutina se debe incluir en SubroutineUniformNames, en la etapa correspondiente (VertexShader, Geometry Shader, Fragment Shader o Compute Shader) y para la subrutina correspondiente.
