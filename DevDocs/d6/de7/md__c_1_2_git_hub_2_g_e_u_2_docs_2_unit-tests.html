Unit-tests

GEU utiliza Google Test como framework de testing. Por tanto, es necesario conocer como escribir un unit test utilizando este framework para escribirlo en GEU. Aquí se puede ver brevemente como escribirlos.

⏮️ Previos

Es necesario tener instalado Test Adapter para Google Test en nuestra instalación de Visual Studio.

:blue_book: Instrucciones

Como crear un unit test

  1. Desde Visual Studio, nos situamos sobre el filtro de Tests, clic derecho → Agregar → Nuevo elemento…
  2. En el lateral izquierdo seleccionamos la categoría de Prueba y aparecerá Google Test. Lo seleccionamos y cambiamos la ubicación a dentro de la carpeta de GEU\Source\Tests.

  3. El fichero comenzará con una plantilla estandar, a la que deberemos añadir un bloque ifdef de la siguiente manera:

    ```cpp #ifdef TESTING #include "gtest/gtest.h" //Dentro de este bloque se definen los tests.

    TEST(TestCaseName, TestName) { EXPECT_EQ(1, 1); EXPECT_TRUE(true); }

    #endif ```

  4. Por último deberemos desactivar los encabezados precompilados para este fichero. Para hacer esto pulsamos clic derecho sobre el fichero en Visual Studio → Propiedades → C/C++ → Encabezados precompilados → No utilizar encabezados precompilados.
    Se tiene que tener cuidado de hacer esto en todas las configuraciones (se puede hacer seleccionando en el desplegable Configuración, arriba a la izquierda de la ventana de propiedades, Todas las config.

  5. Para probar el test solo tenemos que lanzar el programa utilizando la configuración de Testing:

    Cabe destacar que esto lanzará todos los test implementados en la plataforma.

Los test se realizan sin activar el proceso de renderizado. Por tanto, no se podrán probar funcionalidades que dependan de este.
