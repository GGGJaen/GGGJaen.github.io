Cómo actualizar el manual de desarrollador

    Actualizar este manual es un poco más complejo que en el caso del manual de usuario. Estos son los pasos a seguir:

  1. Instalar Doxywizard.
  2. Clonar el repositorio de gggj.github.io y el de GEU.
  3. En el repositorio de gggj.github.io encontraremos un doxyfile prácticamente configurado para generar la documentación. No obstante, es posible que haya que modificar las rutas.
    1. La solución más sencilla para esto es que el repositorio de GEU se encuentre junto al de la web. Dado que las rutas son relativas, esto debería de hacer que funcionara.
    2. Si no se puede seguir el paso 1, no queda otra que modificar el doxyfile, actualizando las rutas. Para esto se debe utilizar el modo experto de Doxywizard. En el caso de hacer esto, no se debe de subir el Doxyfile actualizado, pues eso podría romper el workflow para el resto de compañeros.
  4. Ejecutamos el generador y vemos el resultado. El resultado se deberá guardar en la carpeta DevDocs.
  5. Si todo está correcto, hacemos commit y push, lo que provocará que se despliegue en la web.

En el caso de querer tener una página dedicada a cualquier explicación, como es el caso de esta, debemos incluir un fichero Markdown con el contenido que se desee bajo la carpeta Docs en el repositorio de GEU. Al ejecutar Doxygen, este se encargará de procesarlo e incluirlo en la página final.
